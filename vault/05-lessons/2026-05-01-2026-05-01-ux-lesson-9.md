## Trigger
PRISM family bubble size was scaled by bot count, which meant a family with many bots and no memories (e.g., 110 R-family bots, none with memories) appeared as a large bubble despite having nothing inside, masking the actual memory landscape.

## Lesson
Size the PRISM family bubbles (and the default Prism orb) by **memory volume**, not bot count, on a shared scale. Visual size communicates "how much memory lives here". Keep the disable-on-zero-bots rule for the family button so families with bots-but-no-memories remain tappable and route to the "No memories yet" empty state.
